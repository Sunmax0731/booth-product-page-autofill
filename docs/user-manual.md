# BOOTH Product Page Autofill 利用マニュアル

## できること

この Chrome 拡張は、Markdown で用意した BOOTH 商品ページ原稿を、BOOTH の商品編集画面へ入力します。

入力する項目:
- 商品名
- 商品紹介文の先頭本文
- BOOTH の「段落」ブロック
- 商品画像

入力しない項目:
- タグ
- 価格
- カテゴリ
- 年齢制限
- バリエーション
- 作品ファイル
- 公開設定

## インストール

1. Chrome で `chrome://extensions` を開きます。
2. 右上の「デベロッパー モード」を有効にします。
3. 「パッケージ化されていない拡張機能を読み込む」を押します。
4. このリポジトリのルートフォルダを選択します。

## 入力ファイルを準備する

1. [input-format.md](input-format.md) に沿って Markdown を作ります。
2. `## 商品名` と `## 概要` を必ず入れます。
3. BOOTH の段落にしたい内容は `##` または `###` で分けます。
4. タグは `## タグ案` にメモしても構いませんが、自動入力はされません。
5. 商品画像は PNG / JPG などの画像ファイルとして用意します。

## BOOTH へ入力する

1. BOOTH の商品編集ページ `https://manage.booth.pm/items/<item-id>/edit` を開きます。
2. Chrome ツールバーから `BOOTH Autofill` を開きます。
3. `商品ページ Markdown` に Markdown ファイルを指定します。
4. `商品画像` に商品画像を指定します。
5. ファイル選択ボタンの代わりに、popup 内のドロップ領域へ Markdown と画像をまとめてドラッグ＆ドロップしても読み込めます。
6. プレビューの商品名、画像数、紹介文ブロック数を確認します。
7. `現在の BOOTH 編集ページへ入力` を押します。
8. BOOTH 画面で入力結果を確認します。
9. タグ、価格、カテゴリ、バリエーション、作品ファイルを自分で設定します。
10. 問題なければ BOOTH 側で保存します。

初回は `入力後に保存ボタンを押す` を無効のまま使ってください。入力結果を十分確認できるようになってから有効化します。

## ドラッグ＆ドロップで読み込む

popup 内の「Markdown と画像をここへドロップ」領域にファイルを置くと、自動で種類を判定します。

- `.md`、`.markdown`、`.txt` は Markdown として読み込みます。
- `image/*` のファイルは商品画像として読み込みます。
- Markdown と画像を同時にドロップできます。
- Markdown だけをドロップした場合、既に読み込んだ画像はそのまま残ります。
- 画像だけをドロップした場合、既に読み込んだ Markdown はそのまま残ります。

## 段落の作られ方

Markdown:

```markdown
## 詳細

### 主な特徴

- 軽量な 3D アバターです。
- 表情切り替えに対応しています。

### 内容物

- UnityPackage
- FBX
- Textures
```

BOOTH:
- 段落見出し: `主な特徴`
- 段落本文: 箇条書き本文
- 段落見出し: `内容物`
- 段落本文: 箇条書き本文

`## アップデート履歴` は例外です。中に `### v1.0.0` があっても、BOOTH では `アップデート履歴` という 1 つの段落にまとめます。

## トラブルシュート

商品名が入らない:
- `## 商品名`、`## 商品名案`、`## タイトル` のいずれかを入れてください。
- ない場合は H1 見出しが使われます。

段落が入らない:
- BOOTH 画面に「段落」ブロックを追加できる状態か確認してください。
- BOOTH 側の画面仕様が変わった場合、拡張の DOM 検出調整が必要です。

画像が入らない:
- popup で画像ファイルを選択しているか確認してください。
- BOOTH 側でアップロード処理が終わるまで画面を閉じないでください。

タグが入らない:
- 仕様です。タグは自分で BOOTH に入力してください。
