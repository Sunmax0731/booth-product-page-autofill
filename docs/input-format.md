# BOOTH 商品ページ Markdown 入力フォーマット

このフォーマットは、Chrome 拡張 `BOOTH Product Page Autofill` に読み込ませるための商品ページ原稿です。BOOTH の商品編集画面に直接貼り付ける完成文としても読めるようにしつつ、作業用メモや画像案は自動入力対象から外せる構成にしています。

## 基本構造

```markdown
# <商品名> BOOTH 商品ページ

## 根拠

- Repository:
- Version:
- Confirmed date:
- Source files:

## 商品名

<BOOTH の商品名>

## 概要

<商品紹介文の先頭に入れる短い説明。2-3 文程度。>

## 詳細

### 主な特徴

- <特徴 1>
- <特徴 2>
- <特徴 3>

### 使い方

1. <手順 1>
2. <手順 2>
3. <手順 3>

## 内容物

- <同梱ファイル 1>
- <同梱ファイル 2>

## 対応環境

- <対応ソフト、対応バージョン、対応 OS など>

## アップデート履歴

### vX.X.X

- <変更点>

## 注意事項

- <利用前に確認すべき点>

## タグ案

- <自分で BOOTH に入力するタグ>

## 商品画像構成案

1. <1 枚目の画像案>
2. <2 枚目の画像案>
```

## 自動入力されるセクション

| Markdown | BOOTH への入力 |
| --- | --- |
| `# <商品名>` | 商品名の fallback |
| `## 商品名` / `## 商品名案` / `## タイトル` | 商品名 |
| `## 概要` / `## 商品概要` | 商品紹介文の先頭本文 |
| その他の入力対象 `##` | BOOTH の段落 |
| `###` | BOOTH の段落見出し |
| `###` の本文 | BOOTH の段落本文 |
| 選択した画像ファイル | 商品画像 |

`## アップデート履歴`、`## 更新履歴`、`## バージョン履歴` は特別扱いです。内部に `### v1.2.3` のような版見出しがあっても BOOTH 段落を分けず、1 つの履歴段落にまとめます。

## 自動入力されないセクション

次のセクションは作業用情報として扱い、BOOTH の商品紹介文へは入力しません。

- `## 根拠`
- `## 出典`
- `## タグ案`
- `## タグ`
- `## 商品画像構成案`
- `## 商品画像生成プロンプト`
- `## Chrome 拡張入力フォーマット`
- `## BOOTH入力設定`

タグは自分で BOOTH 画面に入力する前提です。価格、カテゴリ、年齢制限、バリエーション、作品ファイルも拡張では変更しません。

## 商品別の書き方

3D アバター:
- `## 詳細` に「主な特徴」「表情・ギミック」「導入手順」などを置く。
- `## 内容物` に UnityPackage、FBX、Textures、PSD、利用規約などを書く。
- `## 対応環境` に Unity、VRChat SDK、lilToon などの前提を書く。

素材集:
- `## 詳細` に素材の種類、解像度、ファイル形式、使用例を書く。
- `## 内容物` に PNG、PSD、FBX、音声ファイルなどの内訳を書く。
- `## 注意事項` に再配布、商用利用、加工可否などを書く。

ツール、エディタ拡張:
- `## 詳細` に主な機能、作業フロー、想定ユースケースを書く。
- `## 対応環境` に OS、対応ソフト、検証済みバージョンを書く。
- `## アップデート履歴` に直近の購入者向け変更点を書く。
