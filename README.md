# BOOTH Product Page Autofill Chrome Extension

BOOTH の商品編集ページ `https://manage.booth.pm/items/nnnnnnn/edit` で、Markdown の商品ページ原稿と商品画像ファイルを読み込み、商品名・商品紹介文・商品画像を入力する Chrome 拡張です。タグは自動入力しません。

Unity エディタ拡張だけでなく、3D アバター、素材集、PDF、音声素材、ツール、テンプレートなどの BOOTH 商品ページ作成に使えるよう、Markdown の `##` セクションを汎用的な BOOTH 段落として扱います。

## インストール

1. Chrome で `chrome://extensions` を開きます。
2. 「デベロッパー モード」を有効にします。
3. 「パッケージ化されていない拡張機能を読み込む」から、このリポジトリのルートフォルダを選択します。
4. BOOTH の商品編集ページを開きます。

## 使い方

1. 拡張の popup で商品ページ Markdown と商品画像を選択します。
2. popup 内のドロップ領域へ Markdown と画像をまとめてドラッグ＆ドロップしても読み込めます。
3. 「現在の BOOTH 編集ページへ入力」を押します。
4. 商品名、商品紹介文、段落、画像を確認し、問題なければ BOOTH 側で保存します。

`入力後に保存ボタンを押す` を有効にした場合だけ、拡張が保存ボタンのクリックまで試みます。初回利用時は無効のまま、入力結果を目視確認してください。

## 入力される項目

- 商品名: `## 商品名`、`## 商品名案`、`## タイトル` の最初の本文行。なければ H1 見出し。
- 商品紹介文: `## 概要` または `## 商品概要` の本文。
- BOOTH 段落: 入力対象の `##` セクション。`###` がある場合は `###` ごとに段落化。
- 商品画像: popup で選択またはドロップした画像ファイル。

タグ、価格、カテゴリ、年齢制限、バリエーション、作品ファイル、公開保存は自動入力しません。

## ファイルの読み込み

- `商品紹介 Markdown` と `商品画像` のファイル選択ボタンは従来通り使えます。
- popup 内のドロップ領域へ `.md` / `.markdown` / `.txt` と画像ファイルをドロップすると、自動で Markdown と画像に振り分けて読み込みます。
- Markdown だけをドロップした場合は、選択済み画像は維持されます。
- 画像だけをドロップした場合は、選択済み Markdown は維持され、画像だけが差し替わります。

## Markdown の読み取りルール

- `## 根拠`、`## タグ案`、`## 商品画像構成案`、`## 商品画像生成プロンプト` などの作業用セクションは BOOTH 本文へ入力しません。
- `## 概要` は見出しなしの先頭本文として入力します。
- `###` 見出しがあるセクションは、`###` の見出しを BOOTH 段落見出し、続く本文を段落本文として入力します。
- `###` がない入力対象セクションは、`##` 見出しを BOOTH 段落見出しとして入力します。
- `## アップデート履歴`、`## 更新履歴`、`## バージョン履歴` は内部の `### vX.X.X` 見出しで分割せず、1 つの履歴段落本文として入力します。
- 段落フォームが足りない場合は、BOOTH の `+ 段落` ボタンを検出して必要数まで追加します。

詳しい入力ファイル仕様は [docs/input-format.md](docs/input-format.md) を参照してください。

## 実装上の注意

- BOOTH 管理画面の DOM は公開 API ではないため、フォーム検出は label / placeholder / name / id / 周辺テキストからの推定です。
- BOOTH の段落ブロックは `li.js-item-module`、`title` 入力、`content` textarea を優先して検出します。
- 画像入力は最初に見つかった image file input に対して `FileList` を設定します。BOOTH 側のアップロード UI が変わった場合は追加調整が必要です。
- ローカルファイルを拡張から直接パス指定で読むことはできないため、popup で Markdown と画像を選択またはドロップする方式にしています。
